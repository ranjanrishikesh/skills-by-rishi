# Review Framework — Review Mode, Phase 3 Reference

**This phase applies to REVIEW mode only.** In CREATE mode, there is no existing resume to review — the Experience Intake and Resume Architecture phases handle context gathering instead.

Deliver the review under these exact sections. Be specific — quote actual resume content, never give generic advice. Every evaluation must reference the Goal Profile from Phase 1.

---

## 3A. FIRST IMPRESSION AUDIT (The 6-Second Test)

Recruiters spend ~6 seconds on first scan. Evaluate each of these:

### Visual Hierarchy
- Does the most important information (name, target title, key value prop) jump out immediately?
- Is there a clear visual flow: name → title → summary → experience?
- Are section headers distinct and scannable?
- Is the layout cluttered or does it breathe?

### Title & Positioning
- Does the headline/title match the target role from the Goal Profile?
- Would a recruiter searching for [target role] recognize this person as a match in 3 seconds?
- Is the title too generic ("Designer") or appropriately specific ("Senior Product Designer, B2B SaaS")?

### Summary Punch
- Does the first sentence create a reason to keep reading?
- Does it communicate: WHO you are + WHAT you do + WHY you're valuable + FOR WHOM?
- Is it tailored to the goal, or is it a generic "passionate professional" statement?
- Test: Could this summary belong to 100 other people? If yes, it's too generic.

### ATS Keyword Density
- List the top 10 keywords the target role demands (from Goal Profile / JD analysis)
- Check which of those 10 appear in the resume
- Flag which are missing entirely
- Note which appear only in skills but not in experience context (weak signal for ATS)

## 3B. EXPERIENCE SECTION DEEP DIVE

Evaluate EACH role separately. For every role:

### Impact Framing Check
For each bullet, classify as:
- **Outcome-driven** (good): "Reduced production time from 2 weeks to 10 minutes" — has a measurable result
- **Task-driven** (weak): "Responsible for managing the design system" — describes what you did, not what happened
- **Vague** (bad): "Worked on various projects" — no specificity at all

Count: X outcome-driven / Y task-driven / Z vague bullets. Flag the ratio.

### Specificity Audit
For each bullet, ask: "Could I verify this claim?" If the answer is no, it's too vague.
- BAD: "Improved the user experience" — how? by how much? for whom?
- GOOD: "Redesigned onboarding flow, reducing drop-off from 40% to 18% across 12K monthly signups"

### Relevance to Goal Filter
For each bullet, score:
- **HIGH relevance**: Directly maps to a target role requirement
- **MEDIUM relevance**: Tangentially related, shows transferable skill
- **LOW relevance**: Doesn't serve the goal — candidate for removal or rewrite

### Power Verb Analysis
Flag weak openers: "Helped," "Assisted," "Was responsible for," "Worked on," "Participated in"
Suggest strong replacements: "Architected," "Spearheaded," "Reduced," "Launched," "Scaled," "Automated"

### Career Story Arc
- Does the progression tell a story of growth?
- Is there a clear through-line or does it feel random?
- Are promotions / increased scope visible?
- Do transitions between roles make sense?

### Rewrite Showcase
Pick the **3 weakest bullets** from the entire resume. For each:
1. Quote the original
2. Explain why it's weak
3. Rewrite it using the formula: **[Strong Verb] + [What You Did] + [Measurable Result] + [Scale/Context]**

## 3C. SKILLS & TOOLS AUDIT

### Missing Keywords
Cross-reference the Goal Profile's required skills against what's listed:
- Create a two-column table: "Required by Target" vs. "Present in Resume" (Yes/No)
- For each missing skill, note whether the user likely HAS it (just didn't list it) or DOESN'T have it (a real gap)

### Noise Check
Flag any listed skill that:
- Has zero backing evidence in the experience section
- Is too basic to list for the target seniority ("Microsoft Office" for a senior role)
- Is irrelevant to the goal
- Is redundant (listing both the category and specific tools, e.g., "UI Design" + "Visual Design" + "Figma")

### Categorization Review
- Are skills grouped in a way that makes sense for the target role?
- Should categories be renamed to match industry language?
- Is the ordering strategic (most relevant first)?

### Credibility Linkage
For each listed hard skill, check: is there at least one experience bullet that demonstrates this skill in action?
- Skills without evidence = claims without proof. Flag them.

## 3D. STRUCTURAL & FORMATTING REVIEW

### Section Ordering
Current order vs. recommended order for the Goal Profile:
- For experienced hires: Summary → Experience → Skills → Education
- For career pivoters: Summary → Relevant Skills → Experience (reframed) → Education
- For freelancers: Summary → Services/Expertise → Selected Work → Tools → Testimonials

### Length & Density
- Is the resume the right length for the seniority level?
  - 0-5 years: 1 page strongly preferred
  - 5-10 years: 1-2 pages
  - 10+: 2 pages max (unless academic CV)
- Are there bullets that could be combined or cut?
- Is any section disproportionately long?

### Consistency Audit
Check and flag every instance of:
- Inconsistent date formats (May 2023 vs. 05/2023 vs. 2023)
- Inconsistent punctuation (some bullets end with periods, some don't)
- Tense errors (current role should be present tense, past roles past tense)
- Capitalization inconsistencies
- Inconsistent bullet style (dashes vs. dots vs. arrows)
- Spelling variations (British vs. American English — pick one)

### Missing Sections
Based on Goal Profile, should the resume include:
- Projects section? (especially for career pivoters or those with side projects)
- Certifications? (if the target role values them)
- Publications / speaking? (for thought leadership positioning)
- Awards / recognition?
- Volunteer work? (only if strategically relevant)
- Education details? (coursework, thesis — only if recent grad)

## 3E. CONTENT GAPS & MISSED OPPORTUNITIES

Answer these questions specifically:

1. **What achievements are being undersold?** — Identify bullets where the user likely had more impact than they described. Probe for hidden metrics.

2. **What questions would a hiring manager have?** — List 3-5 questions a reader would want answered that the resume doesn't address. These are gaps.

3. **What's the single biggest thing holding this resume back?** — One sentence. Be direct.

4. **What unique angle is being buried?** — Is there a differentiator (unusual background, rare skill combo, impressive achievement) that isn't getting spotlight?
