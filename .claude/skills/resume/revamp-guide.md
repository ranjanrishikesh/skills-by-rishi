# Revamp Guide — Shared Phase Reference

## Mode Context
- **Review mode**: This is Phase 6. You are rewriting the original resume from scratch based on all prior phases. Every change must be traceable to the Goal Profile (Phase 1), must address weaknesses from the Review (Phase 3) and Devil's Advocate (Phase 4), and must incorporate the user's real answers from the Revamp Interview (Phase 5).
- **Create mode**: This is Phase 7 (Refinement). You are refining the draft from the Resume Writer phase based on Devil's Advocate findings and gap-filling answers. Do NOT start over — improve what exists. The Change Log Table maps changes from the draft to the refined version, with each change traced to a Devil's Advocate finding or user answer.

---

This phase delivers a **complete, polished resume**. Not suggestions — a finished product the user can copy-paste and use.

---

## Revamp Principles

1. **Goal-first rewriting**: Every word earns its place by serving the target goal. If a bullet doesn't help the user get closer to their goal, it gets cut or rewritten.

2. **Evidence over claims**: Replace "responsible for X" with "achieved Y by doing X." Every bullet must pass the "so what?" test.

3. **The STAR-lite formula for bullets**:
   ```
   [Strong Action Verb] + [What You Did Specifically] + [Measurable Result or Impact] + [Scale / Context]
   ```
   Examples:
   - BEFORE: "Led design system development to reduce redundancy"
   - AFTER: "Built and scaled a design system of 60+ components adopted by 3 product teams, reducing UI inconsistencies by 40% and cutting design-to-dev handoff time in half"

4. **Keyword integration**: Missing keywords from the Goal Profile / JD analysis must be woven naturally into experience bullets and the summary — not just dumped into the skills section.

5. **Maintain the user's voice**: The revamp should sound like a sharper, more confident version of the user — not a generic template. Preserve their personality and authentic language where it's effective.

6. **Address every Devil's Advocate objection**: Cross-reference Phase 4 findings. Each major weakness should have a corresponding fix in the revamp.

7. **Use Phase 5 answers as primary source**: The user's answers from the Revamp Interview are the most accurate source of truth. Use their real metrics, real context, and real achievements over your own estimates. Only use [VERIFY] for details the user couldn't provide.

## Revamp Structure

Deliver the full resume in this format:

```markdown
# [Full Name]
## [Optimized Title — Tailored to Target Role from Goal Profile]

[Email] | [Portfolio URL] | [LinkedIn URL] | [Location if relevant]

---

### SUMMARY
[3-4 sentences maximum]
- Sentence 1: WHO you are + years of experience + domain (the hook)
- Sentence 2: Your unique value proposition — what you do differently
- Sentence 3: A specific proof point (your best metric or achievement)
- Sentence 4: What you're looking for / alignment to target (only if space allows)

Rules for the summary:
- Must contain at least 2 keywords from the target role
- Must NOT start with "I am" or "Passionate professional"
- Must answer: "Why should I keep reading?"
- Should be specific enough that it couldn't belong to anyone else

---

### EXPERIENCE

**[Company Name]** — [Industry/Domain in parentheses]
**[Optimized Role Title]** | [Start Date] – [End Date or Present]
[Location | Remote]

- [Bullet 1: Highest-impact achievement for this role]
- [Bullet 2: Second-highest impact]
- [Bullet 3: Third, if it serves the goal]
- [Bullet 4: Only if genuinely valuable — prefer 3-4 bullets per role]

Rules for experience bullets:
- Current role: present tense ("Lead..." not "Led...")
- Past roles: past tense
- Start every bullet with a strong action verb (never "Responsible for")
- Include at least ONE metric per role (even estimated ranges are better than nothing)
- Most recent/relevant role gets the most bullets (4-5)
- Oldest/least relevant role gets fewest (2-3)
- Order bullets within each role by relevance to the goal, not chronologically

**[Previous Company]** — [Industry/Domain]
...

---

### SKILLS & TOOLS

Reorganize into categories that match the target role's language:

**[Category 1 — Most Relevant to Goal]**
Skill 1, Skill 2, Skill 3, Skill 4

**[Category 2]**
Skill 5, Skill 6, Skill 7

**[Category 3]**
Tool 1, Tool 2, Tool 3

Rules for skills:
- Lead with the category most relevant to the target
- Remove skills that are too basic for the seniority level
- Remove skills with zero evidence in the experience section (unless they're being added as new)
- Add missing keywords identified in the review
- Use exact terminology from the target JD when possible

---

### [ADDITIONAL SECTIONS — Only if recommended in Phase 3]

Possible additions:
- **Projects**: For side projects, open source, or notable work not covered in experience
- **Certifications**: If the target role values them
- **Education**: Standard format — degree, institution, year. No GPA unless exceptional and recent.
- **Publications / Speaking**: Only if positioning as a thought leader
```

## Metrics Guidance

If the user's original resume lacks metrics, help them think about what to add:

### Metric Prompts by Role Type:
- **Designers**: Users impacted, conversion improvements, time saved, components built, adoption rates, usability test results, NPS changes
- **Engineers**: Performance improvements, uptime, scale (requests/sec, users), time saved, bugs reduced, deploy frequency
- **Product Managers**: Revenue impact, user growth, feature adoption, time-to-market, OKR achievement
- **Marketers**: Traffic growth, conversion rates, CAC reduction, campaign ROI, audience growth
- **General**: Team size led/mentored, cross-functional scope, process improvements, cost savings

If exact numbers aren't available, use:
- Ranges: "50-100 components"
- Approximations: "~30% reduction"
- Comparisons: "cut time from 2 weeks to 10 minutes"
- Scale indicators: "across 3 product teams" or "serving 50K+ users"

Note in the revamp where you've estimated or suggested a metric — mark these with [VERIFY] so the user knows to confirm or adjust the number. Prefer the user's actual answers from Phase 5 over estimates.

This table is critical for:
- Transparency — the user understands WHY each change was made
- Education — teaches the user resume writing principles
- Verification — the user can override changes they disagree with
- Traceability — every change maps to a goal or weakness
- Source tracking — distinguishes user-confirmed facts from estimates

## Positioning Check Before Revamp

IMPORTANT: Before starting the revamp, check for **fundamental positioning problems**:
- Is the user targeting a role that's a realistic next step, or a 3-level jump?
- Is the resume the right artifact? (Maybe they need a portfolio, case studies, or LinkedIn overhaul instead)
- Is there a story mismatch that no amount of bullet rewriting can fix?

If yes — **say so clearly before writing the revamp**. Don't polish a resume that needs a strategic rethink. Offer the rethink first, then revamp based on the corrected positioning.
